毕业论文-简单记录

本着简单记录毕业前的一段时光为目的，随手记录

本文较多引用此知乎论文收藏夹

将从以下几个方面进行记录：

  • 论文选题
  • 撰写开题报告，任务设计书
  • 论文的写作
  • 程序设计
  • 答辩

论文选题

选题部分，要和老师沟通！这点很重要，尤其是有自己的想法和目标之后，提出个人建议：

1、尽量选择自己比较熟悉的领域

2、若是不清楚自己想要做什么，可以去查阅资料（后面说查资料），翻看前人研究内容，做到哪一步了，我将要研究什么

3、确定好创新点，关键技术等

开题报告、任务设计书的撰写

开题报告

有关于开题报告这部分，可以参考这里，万能金句参考这里。开题报告无非几部分，研究背景，国内外研究现状（或立题依据），主要研究内容，设计方案和思路，我相信，把这部分内容描述好了，论文的大体框架就搭的差不多了。研究背景就是为什么要进行这项研究，可从概念，起源等引出研究背景。国内外研究现状，要求我们查阅资料，整理数据，你可以这么写：近年来…目前，国内…，XXX提出…观点，因此…有着重要意义。要对研究现状做一个了解，不能啥也不知道就硬写吧。主要研究内容就是你要研究什么？你可以从以下方面入手：XX的现状是什么，目前存在的不足之处，有什么解决办法，你的创新点是什么？只有搞清楚自己要研究的方向，有一个清晰的认识，才不会乱，别人已经研究过的内容，你还研究有啥意思？关于设计方案和思路，就是大体要研究的计划，怎样才能完成这个计划。这一部分就是划分步骤，比如，把大象放进冰箱有几步？我怎样实现这几步？想要轻松的写出开题报告，阅读大量文献是必不可少的。

任务设计书

任务设计书，就是对主要任务进行划分，任务要求，主要内容，进度安排等，对主要任务进行划分，和设计方案类似，列出主要的任务，目标等等…比如利用XX对XX进行实现，通过XX处理，实现XX操作等，这个过程中可能需要我们涉及新的知识，对于学习能力也有一定的要求。任务要求就是明确你要完成一些研究需要做到什么，主要内容主要研究些什么，进度安排，对于任务进度，合理规划，安排好每一步的时间。

论文的写作

有关于论文的写作，这里先谈一下查资料的问题。我是这样进行的：

  • 利用学校资源，其中知网，维普，万方，超星可以查阅很多文献期刊等
  • 利用校内图书馆资源，如果不清楚借什么样的书籍，可以登录个人图书馆搜索馆藏资源

以上是利用学校有的资源进行查阅，还推荐以下：

  • 谷歌学术（可以利用一些谷歌学术镜像访问）
  • 百度学术
  • 术语在线
  • 爱学术
  • 国家统计局
  • DATA
  • 中经数据
  • ScienceDirect

有很多比较不错的资源可供我们参考，如发现比较不错的资源，我会在评论区贴出^_^

当然，你也可以参考这个回答和这个问题

格式

格式，important！不要像我本篇格式一样开头不缩进，还有注意一下行间距之类的，这是些常见的错误，老师一般会发送一篇已经具备格式范文的论文模板，下面谈一谈我在写作过程中的一些总结

写在前面：我是在老师发的模板基础上改的，所以你也可以看看老师发的模板，在模板基础上尽行论文的写作，那样，格式出错率将大大降低

  • 中文摘要英文摘要

我认为这部分在格式上没啥好说的，就按照模板来就可，模板啥格式，你就啥格式。那怎样写摘要呢？我认为可以从三个方面下手，那就是：目的，方法，结论。列出你研究的目的，例如：近年来…和过去相比…其中XXX(引入目的)，为什么要研究它？不断反问自己，你会有一个答案的。方法怎样写？基于XX讨论了XX探究了XX特别是XX从而…最后来说一下结论，结论就是总结一下，你可以在这里写出缺点和不足，要创新的地方，解决了什么问题，这样做的好处，这样做能够带来什么？从而引入你的结论。英文摘要如果想要优秀，保证语句的通顺和逻辑上的清晰，最好找位英语专业的同学帮帮忙，哈哈哈q(≧▽≦q)

  • 目录

在目录部分，如果在模板基础上进行的话，非常简单，当然也可以自己编辑目录，word引用里面，有目录选项，有手动目录，有自动目录，还可以自定义目录，我们在创建新的标题后，要更新目录，方法：在目录位置鼠标右键，更新域

  • 前言部分

前言部分没写，原因是将前言放到绪论里面了，当成了绪论写。如果要写前言，我有以下建议：

  • 前人关于这方面做了什么研究
  • 本文的研究如何如何
  • 研究的价值意义必要性等
  • 简述论文结构

前言不宜写的过多，应遵从言简意赅，开门见山的原则，让别人明白我们的研究是什么，或起到一个预告的作用，我们从很多书籍中不难发现，一个好的前言对于阅读也是有很大帮助的，如果真的不知道如何去写前言，可以找一篇类似论文，看其前言如何论述。以上就是我对于前言的认识。

  • 绪论

何为绪论？我认为是思绪论述。我在写论文的时候是根据两方面来完成我的绪论内容的，分别是：研究背景和研究意义，课题研究现状（国内现状，国外现状），课题研究的主要内容这三方面，当然，也可以加入论文研究的问题、必要性等等…（也可以顺带一提研究的必要性）要对课题有一个清晰充分的了解，并且像国内外研究现状，这都是要求我们去查阅资料，阅读文献期刊的，这样，你才有内容，有的写，才能把你整个论文的根打牢。我们为什么要去引用？站在巨人的肩膀上，而不是一味的死研究，讲究一个“化劲儿”，把深度提上去。

绪论完成后就可以根据你的目录，整体框架，来构思论文的每部分写作了。我是这样做的，接下来为相关技术介绍，系统需求分析，系统设计，系统实现，总结与展望，目前完成度：

  • 相关技术介绍
  • 系统需求分析
  • 系统设计
  • 系统实现
  • 总结与展望
  • 参考文献
  • 附录
  • 致谢

其中程序代码主要功能基本已完成，剩余工作页面美观优化。继续把我已完成的讲解一下：

这里一提关于论文中画图的问题，我是不太建议粘贴图片的，我建议如果可以的话，还是自己动手画画图，我认为，画图对我们的整体逻辑性有很大的帮助，它不仅有这个好处，在打印时，如果是图片，可能会有模糊不清晰的现象。这里推荐一下微软的Visio，它是一款专业的画图软件，当然，你也可以使用word来进行画图，或者其他画图工具，但我还是建议使用Visio，因为在使用Visio画完图后，放到word中，我们直接就能编辑，如果要修改画好的图，很方便。在一个就是画的图我们应尽量保证它清晰明了，格式尽量保持一致，如线条的粗细，字体的样式等…

咱们继续介绍:

  • 相关技术介绍

作为计算机系的学生，论文写完绪论，写一写相关技术的介绍，把用到的技术栈，使用的架构等介绍一遍，哈哈哈哈。

因为我的毕业设计相对简单，先是介绍了一下使用技术与工具的概况，这里简单一提，就是JSP，Servlet，JDBC等，前端框架Ace Admin（基于Boostrap），数据库MySQL，软件架构模式为MVC模式，应用服务器Tomcat8.5，使用Eclipse开发，Visual Studio Code辅助开发程序和网页，这就是我简单的毕业设计，并没有使用高大上的技术，而是应用比较熟悉的Java Web方面的知识，我认为，你能使用合适的工具把项目做出来，再准确的描述就已经很棒了。再一个就是创新，一定要展现出自己的创新点，自己的想法。

有点小跑题，继续说，我接下来写的是Web的概述，因为我要做的和WEB有关，这里要做一个简要的概述，Web是什么，工作原理等，然后再把它们展开，期间也画了图，画图时要注意，我们为什么要画图？不就是更清晰直观吗。你自己首先要对于自己画的图足够了解，最好顺带解释一下这个图，越详细越好。在格式上，比如，图1-1…XX工作原理如图所示，这时候要把图居中，下面紧跟着文字：图1-1…XX工作原理（也要居中）然后另起一行，用英文表示：Fig1-1… XX working principle（同样设置居中）注意格式，模板什么格式就什么格式。然后接下来就是一些定义像Java技术概述，JSP技术，JSP连接数据库，最后，相关技术写了数据库系统，其中又包括数据库系统简介，MySQL数据库（因为我是我使用到了MySQL数据库，就单独介绍了一下，可以从特点，为何选用等角度出发，进行介绍）

  • 系统需求分析

相关技术介绍完了，来到了系统需求分析，这里不得不提一门学科——软件工程的重要。系统需求，主要用于描述系统具体实现什么，而不是考虑如何实现。我是这样想的：先把你的研究对象介绍介绍，然后进行需求分析，怎么分析呢？可从功能，性能，可行性上进行分析。

  • 系统设计

可从以下几方面入手：系统设计的原则，采用的系统结构，系统分析与设计，系统模块设计，数据库设计，以上是基于我的设计而提出的，可以根据自身情况的不同，进行不同的系统设计

  • 系统实现

这一部分如果程序代码完成了，将很好写，这将是我接下来要进行的的主要内容，打算利用画图工具，画出E-R图，然后制表，（还没做），其余均已完成。

  • 总结与展望

总结与展望就是写写最后论文的总结，展望以下未来，例如，从XX发展来看，今后…，本篇研究的不足之处，希望能够对XX做出贡献…

  • 参考文献

参考文献就是所引用的有关论文，期刊等，怎样在论文中引用呢？使用快捷键Ctrl+Shift++就可以实现论文文献上标，举个例子，在论文的最后，参考文献中做出参考文献的来源

如：樊胜.C/S与B/S的结构比较及Web数据库的访问方式[J].情报科学,2001(04):443-445

我们先选中这一行，进行编号的处理，将编号改为[1].[2].[3]这种形式

从要引用文献的地方，按下快捷键Ctrl+Shift++，这时，我们点击引用，交叉引用，引用类型选择编号项，引用内容选择段落编号（完整上下文），然后点击我们要引用的编号项，这样，论文的参考文献插入编号就实现了。

  • 附录

在附录中，你可以放一些重要的测试结果、图表、程序代码等

  • 致谢

致谢这一部分其实也很好写，主要感谢一些人，首先感谢导师，然后涉及本论文的学者，家人，朋友等…至此论文就写完了。

有关PPT

关于论文答辩PPT，还是简介清晰的原则，不要弄得那么花里胡哨，尽量不用动画，建议还是自己做一下，网上模板看了下，感觉借鉴一下比较好的部分还是可以的。搜一搜学校的校徽、logo等放在其中，可以参考一下学校官网，公众号之类的找找灵感。将所要演讲的大纲内容整出来，分块进行

最好了解一下放映比例，在设计，幻灯片大小中调整，可选择16:9和4:3，也可以自定义大小，如果条件允许，可以提前测试一下

写在最后，本篇文章只是自己的一个简单小小总结，其中有的仅代表个人观点，您在读完本篇文章后可能感叹，“就这？”不知到讲的什么，更加晕了。也可能有所收获，又或者灵光乍现，如果本文能给您带来收获，那想必是极好的；如若不然，我很抱歉给您带来的烦恼。其实在这个过程中，我感觉态度是挺重要的，有人糊弄糊弄就过去了，我认为，最好还是认真写写论文，还是有点收获的，不是吗？在我看来，个体之间的差异性是不同的，其实也没什么，对得起自己让自己不后悔，尽力而为就行，有时候，我们平心而论的问自己真的努力了吗？当结果不尽人意时，好好反思一下吧。冬去春来，寒来暑往，我们随着时光的流逝，也长大了，该加油了，毕竟天道酬勤，不无道理，哈哈哈。最后，希望我们都答辩成功，冲冲冲！
